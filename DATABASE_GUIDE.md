# Database Integration Guide

## Overview

The application now uses PostgreSQL to store candidate resume data and analysis results. When a candidate signs up and uploads their resume, their information is automatically stored in the database.

## Database Schema

### `candidates` Table

| Column | Type | Description |
|--------|------|-------------|
| `candidate_id` | VARCHAR (Primary Key) | Clerk user ID |
| `username` | VARCHAR (Indexed) | Username derived from Clerk email |
| `resume_text` | TEXT | Full OCR-extracted resume text |
| `analysis` | JSON | Complete analysis results including scores, skills, etc. |

The table is automatically created when the backend starts.

## What Gets Stored When a New Candidate Signs Up?

### On Sign Up (via Clerk)
- **In Clerk's system**: Email, password hash, user ID
- **In your database**: Nothing yet (waiting for resume upload)

### On Resume Upload (`/api/upload-resume`)
The following data is saved to the `candidates` table:
- `candidate_id`: Clerk user ID (from JWT token)
- `username`: Extracted from Clerk email (text before @)
- `resume_text`: Full text extracted from resume using GOT-OCR
- `analysis`: JSON object containing:
  - `name`: Parsed from resume
  - `email`: Extracted via regex
  - `position`: Job title/role
  - `classification`: "Strong Fit" / "Trainable Fit" / "Risky Fit"
  - `score`: Overall score (0-100)
  - `experience`: Years of experience
  - `strengths`: Array of identified skills
  - `weaknesses`: Array of missing common skills
  - `technicalScore`: Technical skills score (0-100)
  - `culturalScore`: Cultural fit score (0-100)
  - `growthPotential`: Growth potential score (0-100)

### File System Storage
- Resume file: `resumes/{candidate_id}.{extension}`
- Resume image: `output_images/{candidate_id}.png` (auto-generated by OCR process)

## API Endpoints

### 1. Upload Resume (Modified)
```
POST /api/upload-resume
```
- **Auth**: Required (Clerk JWT)
- **Body**: MultiPart form with resume file
- **Action**: Saves resume, analyzes it, stores everything in database
- **Response**: Analysis results

### 2. Search Candidates (New)
```
GET /api/candidates/search?q={candidate_id_or_username}
```
- **Auth**: Required (Clerk JWT)
- **Query**: `q` - candidate ID or username
- **Action**: Searches database by candidate_id or username
- **Response**: 
  ```json
  {
    "candidate_id": "user_xyz123",
    "username": "johndoe",
    "analysis": { /* full analysis object */ }
  }
  ```

### 3. Get Full Analysis (New)
```
GET /api/candidates/{candidate_id}/full-analysis
```
- **Auth**: Required (Clerk JWT)
- **Access**: Users can only view their own analysis
- **Response**:
  ```json
  {
    "candidate_id": "user_xyz123",
    "username": "johndoe",
    "analysis": { /* full analysis object */ },
    "resume_image_url": "/api/candidates/user_xyz123/resume-image"
  }
  ```

### 4. Get Resume Image (New)
```
GET /api/candidates/{candidate_id}/resume-image
```
- **Auth**: Required (Clerk JWT)
- **Access**: Users can only view their own resume image
- **Response**: PNG image file

## Environment Configuration

Make sure your `backend/.env` has:
```env
DATABASE_URL=postgresql://postgres:password@localhost:5432/karmafit_db
CLERK_JWKS_URL=https://your-clerk-domain/.well-known/jwks.json
```

## Database Setup

### Using Docker (Recommended)
```bash
docker-compose up -d
```

### Manual PostgreSQL Setup
1. Install PostgreSQL
2. Create database:
   ```sql
   CREATE DATABASE karmafit_db;
   ```
3. Update `DATABASE_URL` in `.env`

The `candidates` table will be created automatically on first backend startup.

## Troubleshooting

### Database Connection Error
- Check PostgreSQL is running: `docker-compose ps`
- Verify `DATABASE_URL` in `.env`
- Check database exists: `docker exec -it karmafit-postgres psql -U postgres -l`

### Image Not Loading
- Check `output_images/` folder exists
- Verify resume was processed (check backend logs)
- Ensure CORS is configured correctly

### Analysis Not Found (404)
- User must upload resume first
- Check database has entry: `SELECT * FROM candidates WHERE candidate_id='user_xxx';`
