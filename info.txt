50-61. `verify_token(...)` dependency:
- reads `Authorization` header
- if missing -> 401
- splits into `scheme token`
- scheme must be `bearer` (case-insensitive)
- returns token string.
- catches split errors and returns 401 bad header format.
- Note: token is not cryptographically verified yet (comment states this).

62-65. `/health` endpoint returns `{"status": "healthy"}`.

66-80. `/api/register` endpoint:
- accepts `RegisterRequest`
- calls `register_candidate`
- maps `ValueError` to 400
- maps unknown errors to 500.

81-95. `/api/login` endpoint:
- calls `login_candidate`
- maps `ValueError` to 401
- unknown to 500.

96-141. `/api/upload-resume` endpoint:
- inputs:
  - multipart `file`
  - `candidate_id` form/query value
  - auth token dependency (`verify_token`).
- validates `candidate_id` exists.
- computes lowercase extension from uploaded filename.
- allows only `.pdf .png .jpg .jpeg .bmp .webp .txt .docx`.
- builds save path `resumes/<candidate_id>.<ext>`.
- writes bytes to disk.
- runs OCR text extraction via `read_resume(file_path)`.
- runs analysis with `analyze_resume(resume_text, candidate_id)`.
- returns success payload with file path and analysis.
- preserves explicit `HTTPException` and wraps unexpected errors as 500.

142-176. `/api/resume/{candidate_id}` endpoint:
- requires bearer header via dependency.
- searches for existing stored resume file for candidate across same allowed extensions.
- if none found -> 404.
- if found, re-runs OCR + analysis and returns result.
- wraps errors similarly.

177-180. Script entrypoint to launch Uvicorn on `0.0.0.0:8000`.

helper_funcs.py
1-4. Imports for filesystem and file conversion (`PIL`, `pdf2image`, `docx`).

5-68. `convert_to_image(file_path, output_folder="output_images")`:
- ensures output dir exists.
- extracts file name + extension.
- output is always PNG in output folder.
- if output already exists, returns cached path.
- if input is image: open + save as PNG.
- if PDF: converts first page using Poppler path and saves PNG.
- if TXT: reads text, paints onto white canvas via PIL, saves PNG.
- if DOCX: extracts paragraph text, paints to image, saves PNG.
- else raises unsupported type error.

69-70. Imports `Settings`, creates global `settings` object (loads OCR model at import time).

71-95. `read_resume(resume_path)`:
- reads device/model/processor from settings.
- logs model/device info.
- converts resume to image.
- processor converts image into model tensors.
- sends tensors to model device.
- calls `model.generate(...)` with processor tokenizer and generation options.
- decodes generated tokens excluding prompt portion.
- returns extracted text string.

96-99. Imports regex/json/transformers token-classification pipeline/uuid.
- `json` currently unused.

100-103. Initializes NER tokenizer/model/pipeline (`dslim/bert-base-NER`, aggregation strategy `simple`).

104-225. `analyze_resume(resume_text, candidate_id)`:
- runs NER entities.
- tries first `PER` entity as candidate name.
- regex extracts email.
- regex extracts phone (stored but not used in return).
- regex patterns extract years of experience.
- scans predefined technical keywords from lowercase text to produce strengths.
- de-duplicates strengths and limits to 8.
- attempts role extraction from patterns.
- computes:
  - technical score from number of strengths
  - cultural score from soft-skill keyword matches
  - growth potential from education/certification indicators
  - weighted overall score.
- maps score to classification:
  - >=75 Strong Fit
  - >=50 Trainable Fit
  - else Risky Fit.
- weaknesses = missing items from common skills list (max 5).
- returns structured analysis object with candidate ID + all scores/lists/metadata.

226 onward auth/db section.

226-232. Imports datetime, optional typing, passlib, jose jwt, SQLAlchemy, and os (duplicate import).

233-237. Database setup:
- `DATABASE_URL` from env or default local postgres string.
- creates engine/sessionmaker/base.

238-239. Password context with bcrypt via passlib.

240-243. JWT constants (`SECRET_KEY`, algorithm, expiry minutes).

244-256. SQLAlchemy `User` model:
- table `users`
- `id` UUID string primary key
- unique indexed email
- hashed_password
- optional full_name
- active flag
- created_at
- last_login.

257-258. Auto-creates tables on import (`Base.metadata.create_all(...)`).

259-262. `hash_password` wrapper -> bcrypt hash via passlib.

263-266. `verify_password` wrapper -> bcrypt verify.

267-276. `create_access_token`:
- copies payload
- sets expiry
- adds `exp`
- signs HS256 JWT with secret.

277-346. `register_candidate(...)`:
- enforces min 8-char password.
- opens DB session.
- checks duplicate email.
- generates UUID for user id.
- hashes password.
- inserts user row.
- commits + refreshes.
- creates access token with `sub=email`, `user_id`.
- returns auth payload.
- rolls back on errors; rethrows meaningful exceptions; closes session.

347-401. `login_candidate(...)`:
- opens DB session.
- fetches user by email.
- if no user -> invalid credentials.
- verifies password.
- checks active status.
- updates `last_login`.
- creates JWT token.
- returns auth payload.
- rethrows errors and closes session.
